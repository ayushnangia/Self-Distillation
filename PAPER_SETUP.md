# Paper Training Setup (arXiv:2601.19897)

Extracted from the actual paper LaTeX source (`paper/arxiv.tex`). Use this as ground truth for all experiments.

## Key Facts

- **Model**: Qwen2.5-7B-Instruct (all experiments)
- **Hardware**: Single NVIDIA H200
- **Seeds**: 3 random seeds per config, report mean + 95% CI
- **Selection**: Best VALIDATION checkpoint on target task (not last, not highest test)
- **SDFT cost**: ~2.5x FLOPs, ~4x wall-clock vs SFT
- **Library**: HuggingFace TRL
- **Fine-tuning**: Full (no LoRA/PEFT)

## Datasets

- **Science Q&A**: SciKnowEval Chemistry L-3 subset. 75% train / 5% val / 20% test. Demos generated by GPT-4o (8 samples per prompt, keep 1 correct).
- **Tool Use**: ToolAlpaca, original train-test split. Demos included in dataset.
- **Medical**: HuatuoGPT-o1 English subset. ~20K train, 1000 eval (randomly sampled verifiable questions). Eval by GPT-5-mini judge.
- **Knowledge Acquisition**: 9 Wikipedia articles on 2025 natural disasters (~200K tokens). QA pairs generated by GPT-5. ~5x larger than source.

## Teacher Prompt Template (from Section 3)

```
<Question>
This is an example for a response to the question:
<Demonstration>
Now answer with a response of your own, including the thinking process:
```

Note: colon at end, "including the thinking process" present.

## Evaluation Protocol

- **Greedy accuracy**: temperature=0
- **pass@k**: temperature=1.0, top_p=0.95, k up to 128
- **Benchmarks**: HellaSwag, TruthfulQA, MMLU, IFEval, Winogrande, HumanEval (via lm-evaluation-harness)
- **Medical judge**: GPT-5-mini, outputs CORRECT/INCORRECT
- **3 seeds**, mean + 95% CI

## Sequential Continual Learning

- Task order: Science → Tool Use → Medical
- Single model trained sequentially on each task
- Performance linearly normalized (0=base, 1=max)
- SDFT maintains all tasks; SFT shows severe interference

### Key Discrepancies

1. **Our base is 12pp higher** (54.4 vs 42.9) — different eval data? different eval protocol?
2. **Our SFT retains prior caps much better** (64.6 vs 56.0 avg) — paper's SFT forgets way more
3. **Our SDFT at 1 epoch (55.9%) is below paper's (70.6%)** — we used lr=2e-5 not in their grid
4. **Our SDFT at 8 epochs (64.7%) still below paper** — and with massive forgetting (58.7 avg)
5. **Our benchmarks differ** — especially TruthfulQA (64.8 vs 47.9 base) and IFEval (57.3 vs 74.3)

### Possible Explanations

- Different eval data split (we use upstream Arrow data, paper may use different split)
- Different lm-eval version / task configurations
- We haven't swept hyperparameters — paper picks best from 54+ configs
- We only ran 1 seed
- Our SDFT was trained at lr=2e-5 (not in paper's grid)
